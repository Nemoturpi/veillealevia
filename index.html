FAQ - Questions fréquentes
À quoi sert cet écran ?

Il sert à relire les informations trouvées par la veille, décider ce qui mérite d'être gardé, mettre certains signaux En attente, ajouter du contexte manuel, puis préparer le prompt du prochain bulletin.

D'où viennent les informations ?

Les informations viennent des sources configurées dans BriefLayer. Les workflows collectent les flux, normalisent les contenus, les scorent par rapport au client, puis les stockent dans Supabase. Cette page sert d'interface de validation.

Comment lire les scores ?

Chaque information reçoit un score de 0 à 100 qui mesure la solidité du lien avec le périmètre de veille du client, accompagné d'un « Pourquoi » : la justification en une phrase.

80+ · Prioritairele lien avec le périmètre est explicite et direct.
60-79 · Pertinentlien réel mais indirect : c'est là que le regard de consultant fait la différence.
40-59 · Signal faiblele lien n'a pas pu être établi clairement, sans pouvoir être exclu. Ces cartes se traitent via l'étape « Trier les signaux faibles ».
URGENTsignal temporel ou sensible détecté en amont. Il attire l'œil, mais ne remplace pas le tri par score.

En dessous de 40, l'information est ignorée automatiquement et n'apparaît pas dans la pile de revue.

Pourquoi une information du jour peut ne pas apparaître ?
  • Elle est sous le seuil de 40 et a été écartée automatiquement.
  • Elle est en Zone grise et n'a pas encore été chargée via l'étape dédiée.
  • Elle est hors période dans le filtre actif : aujourd'hui, 3 derniers jours ou tout.
  • Elle a déjà été ignorée, mise En attente, ou rattachée à un bulletin généré.
  • Le workflow de collecte ou de scoring du jour n'a pas encore fini de tourner.
Que veut dire Garder / Ignorer ?

Garder signifie que l'information mérite une décision éditoriale. Ignorer signifie qu'elle ne sera pas utilisée pour ce bulletin. Ignorer n'efface pas la source : cela change son statut dans la base.

Que veut dire À arbitrer ?

À arbitrer regroupe les cartes gardées lors de la première étape, mais pas encore envoyées vers le Prochain bulletin ni mises En attente. C'est la zone de second choix.

Quelle différence entre Prochain bulletin et En attente ?

Prochain bulletin signifie que la carte sera utilisée dans le prompt de génération. En attente signifie que l'information reste intéressante, mais qu'elle est reportée pour plus tard et ne part pas dans le bulletin en cours.

À quoi servent les informations supplémentaires ?

Elles permettent d'ajouter une note interne, un retour terrain, une source hors flux ou un point de contexte. Elles sont incluses dans le prompt du prochain bulletin du client actif.

Que fait Générer le bulletin ?

Le bouton crée un prompt complet à partir des cartes en Prochain bulletin et des informations supplémentaires. Il copie le prompt dans le presse-papier et le garde dans l'historique des bulletins.

Les décisions sont-elles partagées entre utilisateurs ?

Les décisions principales, comme Garder, Ignorer et Prochain bulletin, sont enregistrées dans Supabase : elles sont donc partagées entre les personnes qui consultent la même base. Les éléments locaux du navigateur, comme certaines infos manuelles ou la file En attente selon le paramétrage actuel, peuvent rester propres au poste utilisé.

Pourquoi une source peut être marquée comme référence interne ?

Quand une carte pointe vers retrotest.local, BriefLayer, un domaine .local ou une référence interne, ce n'est pas une source primaire publiable. La carte doit alors être vérifiée et reliée à la vraie source avant d'entrer dans un bulletin client.

Que faire si je me trompe ?

Une carte peut être reprise depuis les boutons d'action quand elle est encore visible. Le bouton « Réinitialiser » remet la revue du client à l'état initial : à utiliser avec prudence, surtout si plusieurs personnes travaillent sur le même client.

Que fait Dernier bulletin ?

Il rouvre le dernier prompt généré pour le client actif. C'est utile pour retrouver ce qui a été envoyé à l'IA, vérifier les cartes utilisées, ou rouvrir la pile si le bulletin doit être repris.

Prompt du bulletin —

Le prompt est copié : collez-le dans Claude (ou votre IA habituelle), répondez à ses questions, ajoutez votre édito. Les cartes « prochain bulletin » et les infos supplémentaires du client actif sont utilisées.
