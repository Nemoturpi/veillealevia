FAQ - Questions fréquentes
À quoi sert cet écran ?

Il sert à préparer un bulletin de veille client : trier les informations, retenir celles qui doivent entrer dans le bulletin, ajouter du contexte si besoin, puis générer une base de rédaction.

Dans quel ordre traiter la veille ?

Commencez par l'étape 1 pour garder ou ignorer les informations principales. Passez ensuite aux signaux faibles si vous voulez examiner les informations moins évidentes. À l'étape 3, choisissez ce qui entre dans le bulletin. À l'étape 4, ajoutez seulement les compléments utiles. L'étape 5 sert au dernier contrôle avant génération.

Comment lire les scores ?

Le score est une aide au tri, pas une décision automatique. Plus il est élevé, plus l'information semble proche du périmètre du client.

80+ · Prioritaireà regarder en premier : le lien client est fort.
60-79 · Pertinentsouvent utile, mais demande une lecture éditoriale.
40-59 · Signal faibleà examiner si vous cherchez des signaux moins évidents ou émergents.
URGENTattire l'attention sur un sujet potentiellement sensible ou daté.
Quelle différence entre Garder et Ignorer ?

Garder signifie que l'information mérite d'être relue à l'étape suivante. Ignorer signifie qu'elle ne semble pas utile pour la veille en cours. Une information ignorée n'est pas un jugement définitif sur la source : c'est seulement une décision pour ce traitement.

Que faire des signaux faibles ?

Les signaux faibles sont des informations moins directement reliées au client. Ils peuvent être ignorés rapidement, mais ils sont utiles quand un sujet émergent, une tendance faible ou un angle indirect mérite d'être surveillé.

Quelle différence entre Retenir pour le bulletin et Reporter ?

Retenir pour le bulletin signifie que l'information doit alimenter la génération du bulletin courant. Reporter signifie qu'elle reste intéressante, mais qu'elle ne doit pas entrer dans ce bulletin sans nouvelle décision.

Quand ajouter une info supplémentaire ?

Ajoutez une info supplémentaire uniquement si elle apporte un contexte que les informations retenues ne contiennent pas : retour terrain, échange récent, précision client, point de vigilance, source hors veille ou consigne éditoriale.

Que vérifier avant de générer ?

Vérifiez que les informations retenues sont bien celles du bulletin courant, que les dates sont compréhensibles, que les sources primaires sont utilisables, et que les infos reportées ne contiennent rien d'indispensable pour ce bulletin.

Que fait Générer le bulletin ?

Le bouton prépare le texte de travail à donner à l'outil de rédaction. Il rassemble les informations retenues, les compléments saisis à la main et les consignes utiles pour produire un bulletin cohérent.

Que faire si une source est marquée à vérifier ?

Ne l'utilisez pas telle quelle dans un bulletin client. Ouvrez ou recherchez la source primaire, vérifiez le contenu, puis reformulez avec prudence si l'information est conservée.

Que faire si je me trompe ?

Utilisez les boutons de l'information ou la vue liste pour la déplacer dans la bonne colonne. Le bouton de réinitialisation de l'étape sert à revenir en arrière sur l'étape en cours.

Que fait Dernier bulletin ?

Il permet de rouvrir la dernière base de bulletin préparée pour le client actif, par exemple pour relire ce qui avait été retenu ou reprendre une génération récente.

Base de bulletin —

Le texte de travail est copié : collez-le dans Claude (ou votre IA habituelle), répondez à ses questions, ajoutez votre édito. Les infos retenues pour ce bulletin et les infos supplémentaires du client actif sont utilisées.
